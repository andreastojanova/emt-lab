# 📚 Лабораториска вежба 1 (2026) – Група А

## 📌 Барања за изработка

Потребно е да се изработи **API за изнајмување на книги** од одредена библиотека со неколку функционалности како:

- додавање нова книга којашто може да се изнајми;
- ажурирање на постоечки запис за одредена книга;
- бришење книга која веќе не е во добра состојба и не може да се изнајмува; и
- обележување книга како изнајмена.

За оваа лабораториска вежба потребно е да креирате соодветна **Spring Boot апликација**, односно **REST API**.  

Во однос на базата на податоци, потребно е да користите:

- **PostgreSQL**
- **Flyway** за управување со миграциите.

---

# 📖 Податоци за книгите

Во рамки на апликацијата се чуваат следните податоци за книгите:

| Поле | Тип |
|-----|-----|
| id | Long |
| createdAt | LocalDateTime |
| updatedAt | LocalDateTime |
| name | String |
| category | enum |
| author | Author |
| state | enum |
| availableCopies | Integer |

# 🏗️ Архитектура

Користејќи **слоевита архитектура** треба да се дефинираат соодветни:

- **Модели (Entities)**
- **Репозиториуми (Repositories)**
- **Сервиси (Services)**
- **Контролери (Controllers)**

---

# 🗄️ База на податоци

За секој од ентитетите потребно е да внесете **иницијални податоци** преку **Flyway миграција**.

---

# 📦 DTO и Валидација

За **DTOs** потребно е да дефинирате **валидација** таму каде што сметате дека е соодветно.

---

# ⚠️ Справување со грешки

За справување со грешките што можат да настанат при извршување потребно е да дефинирате **соодветен exception handler**.

---

# 📑 API Документација

Со помош на **SwaggerUI** потребно е да прикажете **успешно извршување на API повиците**.

---

# 💡 Забелешка

Секаде каде што проценувате дека е потребно може да ја **надградите апликацијата со дополнителни елементи** што не се експлицитно спомнати погоре, а се покриени во **материјалите од аудиториски вежби**.
